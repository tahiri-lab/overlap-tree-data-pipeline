
# Reference-tree-based datasets of phylogenetic trees defined on different but overlapping sets of taxa (Mode 2)

This folder contains the benchmark datasets constructed by the proposed data pipeline in Mode 2. In this construction mode, each dataset includes both a fixed **reference tree** and a collection of overlapping **phylogenetic trees** obtained by pruning the reference tree to partially intersecting taxon subsets.

## Data description :deciduous_tree:

The Mode 2 datasets are intended for benchmarking phylogenetic methods in settings where direct comparison to a known target tree is required. For each species group, a broad-coverage phylogenetic tree [downloaded from VertLife](https://data.vertlife.org/) was used as the starting tree. A sequence of 20 target taxon subsets was generated, with target sizes ranging from **50 to 145 taxa** in increments of 5. Each target subset was used to prune the starting tree and obtain one **reference tree**. From each reference tree, a corresponding set of **30 overlapping input trees** was generated by pruning to partially intersecting taxon subsets.

For the released datasets, the overlap design is the same across all four groups:

- 20 datasets per species group
- 1 reference tree per dataset
- 30 input trees per dataset
- Pairwise taxon overlap constrained using Jaccard similarity in the range 0.30--0.70
- Full taxon coverage enforced within each dataset
- 10 anchor taxa included in every input tree
- Controlled perturbation applied only to the input trees:
  - 3 nearest-neighbor interchange (NNI) moves per tree
  - Anchor taxa protected from being moved
  - Branch lengths scaled by a single multiplicative factor sampled uniformly from 1.003 to 1.009

This directory contains four species-group subfolders:

- `amphibians_datasets/`
- `mammals_datasets/`
- `sharks_datasets/`
- `squamates_datasets/`

Within each species-group folder:

- `multiset_1.txt` to `multiset_20.txt` contain the 30 overlapping input trees for each dataset in Newick format
- the corresponding `*_reference_trees20.txt` file contains the 20 reference trees, one for each dataset

## Validation

In addition to the prepared datasets, this folder also contains the script used to validate the datasets and the validation results. The audit verifies:

- input-tree taxon counts
- pairwise Jaccard overlap across all unordered tree pairs
- minimum number of shared taxa across tree pairs
- presence of the shared anchor core
- full taxon coverage relative to the reference tree
- successful parsing of Newick strings
- absence of duplicate leaf labels within a tree
- strictly positive branch lengths after scaling

All released datasets passed the checked constraints, with:

- 0 full-coverage failures
- 0 subset failures
- 0 duplicate-label failures
- 0 non-positive branch-length failures
